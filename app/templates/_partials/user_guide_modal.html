{# ── Important notice ── #}
Please stop or pause your VM when you're done working.

Running VMs consume resources on shared Mac hosts even when you're not using them. Always Stop your VM at the end of your session. If you want to keep your work, use Save & Shutdown to archive it safely.

{# ── Overview ── #}
What is MAppleLab?

MAppleLab lets you create, run, and manage macOS virtual machines from your browser. You can open a live console, save a VM to resume later, and manage your VMs within your assigned resource limits.

{# ── Dashboard ── #}
My VMs

The My VMs page is your home. It lists all your VMs and their current status. The table refreshes automatically every few seconds.

{# ── Create VM ── #}
Creating a VM
  1. Click Create VM in the navigation bar
  2. Choose a base image from the dropdown (Gold Images are admin-prepared; Base Images are standard)
  3. Optionally set CPU and memory
  4. Give your VM a name and click Create VM
  5. Wait for the status to change from creating to running — this may take a few minutes on first use

Your quotas limit how many VMs you can run simultaneously and how many you can keep saved.

{# ── Start / Stop ── #}
Starting & Stopping
  • Stop — shuts the VM down. It stays on its node and can be started again instantly.
  • Start — powers a stopped VM back on.
{# ── Save & Resume ── #}
Saving & Resuming
  • Save & Shutdown — stops the VM and uploads its full disk to the registry. Status becomes archived. Use this to free up node resources while keeping your VM.
  • Resume — downloads the archived VM back to a node and starts it. Status goes archived → pulling → running.

Saving can take several minutes depending on disk size.

{# ── Console ── #}
Accessing the Console

The VM must be running to open a console.

  • Console — opens a browser-based noVNC session. Requires HTTPS for remote access.
  • VNC (macOS) — downloads a .vncloc file. Open it with macOS Screen Sharing for a native experience.

If the console fails to connect, stop and restart the VM then try again.
